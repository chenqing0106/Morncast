1.1 产品定位
赛道：赛道二 · 信息流
一句话：你稍后再看里的视频，你真的会回去看吗？我们替你看完，变成你通勤时的播客。
核心洞察：
- 用户在抖音收藏/稍后再看的内容，是经过主动筛选的高质量内容
- 但「有空再看」永远不会发生——视频这个格式要求你盯着屏幕
- 通勤时耳朵是空闲的，眼睛不是。
Morncast 做的事：把你已经认可但没时间消费的内容，转换成你能消费的格式。

1.2 目标用户

核心用户：信息焦虑的通勤者
- 25–35 岁，一二线城市
- 重度抖音用户，关注知识类/行业类博主（AI、创业、设计、财经）
- 每天通勤 30–60 分钟，习惯用耳机
- 收藏夹/稍后再看里积压了大量视频，几乎从不回去看
- 想把碎片时间转化为有效信息摄入，但现有播客跟不上自己的兴趣
核心痛点一句话：
我知道那些视频有价值，但我永远找不到一个完整的时间去看。


---
1.3 用户旅程
你平时刷抖音
    看到感兴趣的视频 → 收藏 / 稍后再看
    （这个动作代表：这个内容值得我消费）

夜间，系统自动处理你的积压内容
    提取字幕 → 按话题聚类 → 生成播客脚本 → TTS 合成
    
出门通勤，戴上耳机，打开抖音
    播客已经在等你
三个关键体验原则：
- 内容是用户自己认可过的，不是算法强推的
- 用户醒来时内容已就绪，无需等待
- 收听发生在放下手机之后（通勤）
